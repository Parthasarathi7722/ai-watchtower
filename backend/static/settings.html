Authentication

Configure how users are authenticated

Current Mode
Local Auth Active by default
1
Set credentials in .env
AUTH_MODE=local
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=<bcrypt-hash>
2
Generate a bcrypt hash
python3 -c "import bcrypt; print(bcrypt.hashpw(b'your-password', bcrypt.gensalt()).decode())"
3
Manage additional users via this Settings page
Once deployed, use the Users tab to add teammates — no restart needed.
AWS Secrets Manager Production recommended
1
Create a secret in AWS Secrets Manager
aws secretsmanager create-secret \
  --name "ai-watchtower/credentials" \
  --secret-string '{"admin": "$2b$12$YOUR_HASH"}'
2
IAM permission required
{
  "Effect": "Allow",
  "Action": ["secretsmanager:GetSecretValue"],
  "Resource": "arn:aws:secretsmanager:*:*:secret:ai-watchtower/*"
}
3
Set in .env and restart
AUTH_MODE=secrets_manager
AUTH_SECRET_NAME=ai-watchtower/credentials
⬡ SSO — Okta / AWS Cognito (coming soon)
Set AUTH_MODE=okta or AUTH_MODE=cognito to enable SAML / OIDC federation. The platform will redirect unauthenticated users to your IdP login page. Configure by setting SSO_CLIENT_ID, SSO_CLIENT_SECRET, and SSO_DOMAIN in .env.

The Users tab will show SSO-synced users once provisioning is configured.
Gate Thresholds

Maximum % of adversarial probes an agent may fail per category before being blocked. Configure via environment variables and restart.

Zero-tolerance categories Block on any failure
Permissive categories Configurable tolerance
To change thresholds: update the corresponding env var in .env (or docker-compose.yml) and restart the API and worker containers.
Example: PROMPT_INJECTION_THRESHOLD=0, PII_LEAK_THRESHOLD=5, CONTENT_VIOLATION_THRESHOLD=10
Galactus AI Configuration

AI-powered security intelligence — AWS Bedrock or Bedrock AgentCore

Current Status
Bedrock Direct (Converse API) Works out of the box
1
IAM permission required
{
  "Effect": "Allow",
  "Action": ["bedrock:InvokeModel", "bedrock:Converse",
             "bedrock:ListFoundationModels"],
  "Resource": "*"
}
2
Enable cross-region inference (for us.* model IDs)
AWS Console → Amazon Bedrock → Model Access → Enable cross-region inference profiles. Or use anthropic.claude-3-5-sonnet-20241022-v2:0 for on-demand only accounts.
3
Set model in .env
GALACTUS_MODE=bedrock
GALACTUS_MODEL_ID=us.anthropic.claude-sonnet-4-6
Bedrock AgentCore Persistent memory + tools
1
Create an Agent in AWS Console
AWS Console → Amazon Bedrock → Agents → Create Agent. Set the system prompt to the Galactus security expert persona. Attach Action Groups for scan triggers and Knowledge Base for OWASP docs.
2
IAM permission for AgentCore
{
  "Effect": "Allow",
  "Action": "bedrock-agent-runtime:InvokeAgent",
  "Resource": "arn:aws:bedrock:*:*:agent-alias/AGENT_ID/ALIAS_ID"
}
3
Set in .env
GALACTUS_MODE=agentcore
GALACTUS_AGENT_ID=<agent-id>
GALACTUS_AGENT_ALIAS_ID=<alias-id>
